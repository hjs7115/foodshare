# 반띵 소프트웨어 설계서 수정 사항

기준 문서: `반띵_허준서_강신혁_소프트웨어설계서 (3).pdf`

## 1. 프로젝트 명칭

- 문서 및 시연 명칭은 `반띵`으로 통일한다.
- 기존 설명에서 `Foodshare`, `푸드쉐어`로 표현된 부분은 `반띵` 또는 `지역 기반 식재료 나눔 및 공동구매 플랫폼`으로 변경한다.
- GitHub 저장소명이나 DB 스키마명처럼 내부 식별자로 쓰이는 `foodshare`는 유지할 수 있다.

## 2. 데이터 저장 방식

설계서에 다음 내용을 보강한다.

- 게시글, 게시글 이미지, 관심 목록, 내 게시글, 프로필 정보는 백엔드 API와 DB를 기준으로 관리한다.
- `localStorage`는 인증 토큰, 자동 로그인 여부, 사용자 정보 캐시, 위치 설정, 알림 설정 같은 보조 정보 저장에만 사용한다.
- 게시글 작성 후 화면 이동 또는 새로고침 시에도 서버에서 게시글 목록을 다시 조회하여 카드와 이미지가 유지되도록 한다.

## 3. 이미지 처리 흐름

이미지 업로드 설명을 다음 흐름으로 수정한다.

1. 사용자가 게시글 또는 프로필 이미지를 선택한다.
2. 프론트엔드는 `POST /api/uploads/images`로 이미지를 먼저 업로드한다.
3. 백엔드는 `{ imageUrl }`을 반환한다.
4. 프론트엔드는 반환된 `imageUrl`을 게시글의 `imageUrl` 또는 사용자의 `profileImage`로 저장한다.
5. 화면에서는 `/uploads/...` 경로를 백엔드 서버 주소와 결합해 이미지를 표시한다.

## 4. 게시글 응답 필드 추가

`GET /api/posts`, `GET /api/posts/{postId}` 응답에 작성자 프로필 이미지를 추가한다.

```json
{
  "postId": 1,
  "writerId": 1,
  "writerNickname": "작성자",
  "writerProfileImage": "/uploads/profile.png",
  "postType": "SHARE",
  "title": "곰 하나",
  "imageUrl": "/uploads/post.png"
}
```

## 5. 관심 목록 API 보강

관심 등록/해제 외에 관심 상태 조회 API를 추가한다.

| Method | Endpoint | 설명 | Response |
| --- | --- | --- | --- |
| GET | `/api/posts/{postId}/favorites/status` | 현재 사용자의 관심 등록 여부 조회 | `{ favorite, isFavorite }` |

## 6. 거래 요청 API 정리

설계서의 거래 요청 API는 현재 구현과 맞춰 다음처럼 정리한다.

| Method | Endpoint | 설명 |
| --- | --- | --- |
| POST | `/api/posts/{postId}/trade-requests` | 거래 요청 생성 |
| GET | `/api/posts/{postId}/trade-requests` | 게시글별 거래 요청 조회 |
| GET | `/api/mypage/trade-requests` | 내가 보낸 거래 요청 조회 |
| GET | `/api/mypage/received-trade-requests` | 내가 받은 거래 요청 조회 |
| POST/PATCH/PUT | `/api/trade-requests/{tradeRequestId}/accept` | 거래 요청 수락 |
| POST/PATCH/PUT | `/api/trade-requests/{tradeRequestId}/reject` | 거래 요청 거절 |
| POST/PATCH/PUT | `/api/trade-requests/{tradeRequestId}/complete` | 거래 완료 처리 |

## 7. 화면 흐름도 수정

현재 시연 흐름에 맞춰 UI/UX 설계의 화면 흐름도를 다음 순서로 수정한다.

```text
초기 화면
  -> 회원가입
  -> 로그인
  -> 카테고리 선택 화면
  -> 메인 게시판
  -> 게시글 작성
  -> 카드 생성 및 상세 확인
  -> 내정보/설정
```

설정 및 마이페이지 내부 기능에는 다음 항목을 포함한다.

- 프로필 조회 및 수정
- 프로필 이미지 변경
- 관심 목록 조회 및 해제
- 내 게시글 조회 및 삭제
- 위치 설정
- 알림 설정
- 거래 내역 및 받은 거래 요청 조회

## 8. 테스트 케이스 추가 및 수정

설계서의 테스트 케이스에 다음 항목을 추가한다.

| TC-ID | 테스트 항목 | 절차 | 기대 결과 |
| --- | --- | --- | --- |
| TC-009 | 이미지 첨부 게시글 유지 | 이미지 첨부 게시글 작성 -> 다른 화면 이동 -> 메인 게시판 재진입 | 서버에 저장된 이미지가 카드에 유지된다. |
| TC-010 | 프로필 이미지 반영 | 프로필 이미지 변경 -> 게시글 상세 화면 접속 | 작성자 프로필 이미지가 변경된 이미지로 표시된다. |
| TC-011 | 서버 기반 데이터 유지 | 게시글 작성 -> 새로고침 또는 화면 이동 -> 게시글 목록 조회 | localStorage가 아닌 서버 조회 결과로 게시글이 표시된다. |
| TC-012 | 거래 요청 생성 | 게시글 상세 화면 -> 요청 버튼 클릭 | 거래 요청 API가 호출되고 요청 생성 결과가 반환된다. |

## 9. 현재 구현 기준으로 보류 또는 축소할 항목

- 별도 냉장고 형태의 `식재료 등록 및 관리` 기능은 현재 게시글 작성 기능 중심으로 구현되어 있다.
- 문서에서 FR-002를 유지하려면 별도 식재료 관리 화면과 API가 필요하다.
- 현재 범위 기준으로는 FR-002 설명을 `게시글 기반 식재료 등록 및 관리`로 축소하는 것이 적절하다.
